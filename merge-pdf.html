How merging works (technical)

We use pdf-lib to copy page objects from each file into one output PDF in your chosen order. We do not re-encode page content in normal operation — quality and fonts stay as in the sources, within browser memory limits.

FAQ

Frequently Asked Questions

How many PDF files can I merge at once?
You can merge as many PDFs as your browser's memory allows. For typical use, merging 2–20 files works smoothly. Very large PDFs (100+ pages each) may require a few seconds to process. There is no hard limit on the number of files.
Can I reorder the PDF files before merging?
Yes! Each file in the list has Move Up (↑) and Move Down (↓) buttons to change the order. Files are merged top-to-bottom in the order shown in the list. You can also remove individual files using the ✕ Remove button.
Is my PDF uploaded to any server?
No. All PDF processing uses the pdf-lib JavaScript library running entirely in your browser. Your files never leave your device and are never uploaded to our servers — completely private processing.
Can I merge password-protected PDFs?
Password-protected PDFs cannot be merged without being decrypted first. If you encounter an error for a protected PDF, use a PDF Decryption tool to remove the password first, then add the decrypted version to the merger.
Will merging preserve the original PDF quality?
Yes. This tool copies all pages from source PDFs to the output document without re-encoding or compressing the content. The merged PDF preserves the original quality, embedded fonts, vector graphics, and images at full fidelity.
